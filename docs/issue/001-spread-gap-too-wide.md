# Issue 001: 見開き中央のギャップが広すぎる

## 概要

見開き表示（スプレッドモード）において、左ページと右ページの間に大きな隙間が生じている。

## 原因

`style.css` の `.page-slot` に `justify-content: center` が設定されているため、
左スロット・右スロットそれぞれの中でページ画像が**水平方向に中央揃え**される。
その結果、左スロットの画像は右端から離れ、右スロットの画像は左端から離れるため、
画面中央（綴じ部分）に不自然な余白が発生する。

```
| [左スロット: 中央寄せ] | [右スロット: 中央寄せ] |
       ← 余白 → [画像]         [画像] ← 余白 →
               ↑ここに大きな隙間
```

## 修正内容

左スロット（`#slot-left`）は画像を**右寄せ**（`justify-content: flex-end`）、
右スロット（`#slot-right`）は画像を**左寄せ**（`justify-content: flex-start`）に変更する。
これにより、両ページが綴じ部分に向かって詰まり、自然な見開きに見える。

```
| [左スロット: 右寄せ]   |  [右スロット: 左寄せ]  |
               [画像] → | ← [画像]
                        ↑ 綴じ部分が自然に密着
```

### 変更ファイル

- `style.css`

### 差分

```css
/* 変更前 */
.page-slot {
  justify-content: center;
}

/* 変更後 */
.page-slot {
  justify-content: center; /* デフォルト（単ページ表示時などのフォールバック） */
}

#slot-left  { justify-content: flex-end;   }
#slot-right { justify-content: flex-start; }
```

## 確認方法

1. 見開き2ページが表示されている状態で、中央の綴じ部分の隙間が目立たなくなっていること
2. 左右どちらか片方だけ表示される場合（奇数ページ先頭など）は、空スロット側が `empty` クラスになるため見た目への影響なし
